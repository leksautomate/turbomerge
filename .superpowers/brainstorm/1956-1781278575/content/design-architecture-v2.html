Section 1 of 5 — Multi-provider generation

Project Structure
bulk-media-generator/
├── app/
│   ├── (dashboard)/     # UI pages
│   └── api/             # REST endpoints
├── worker/
│   ├── index.ts         # Starts pg-boss, registers handlers
│   ├── image-worker.ts  # Routes to Runware or Vertex Image
│   ├── video-worker.ts  # Vertex AI Veo, polls operation ID
│   └── providers/
│       ├── runware.ts     # openai:gpt-image@2 via Runware
│       ├── vertex-image.ts # Imagen 4 via Vertex AI
│       └── vertex-video.ts # Veo via Vertex AI
├── lib/
│   ├── db.ts            # Prisma client
│   ├── queue.ts         # pg-boss instance
│   └── storage.ts       # File save / zip helpers
├── prisma/schema.prisma
└── uploads/
    ├── images/
    ├── videos/
    └── zips/
Worker routing — image jobs
pg-boss fires "image-job"
  → image-worker reads job.model
  → "runware"   → providers/runware.ts    → Runware API
  → "vertex"   → providers/vertex-image.ts → Vertex Imagen 4
  → saves file to uploads/images/:projectId/
  → updates job: status=completed, output_path
Worker routing — video jobs
pg-boss fires "video-job"
  → video-worker → providers/vertex-video.ts
  → POST to Vertex AI → get operation_id
  → saves operation_id to job row
  → polls every 30s until complete
  → downloads MP4 → uploads/videos/:projectId/
  → updates job: status=completed
Per-project model settings
// Image project — Runware
{ "provider": "runware", "model": "openai:gpt-image@2", "quality": "high", "format": "png" }

// Image project — Vertex
{ "provider": "vertex", "model": "imagen-4", "aspectRatio": "16:9", "format": "png" }

// Video project — Vertex Veo
{ "provider": "vertex", "model": "veo", "duration": 8, "resolution": "1080p", "aspectRatio": "9:16" }
