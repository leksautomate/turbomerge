Design: Architecture Overview

Section 1 of 5 — How everything fits together

Project Structure
bulk-media-generator/
├── app/                  # Next.js pages + API routes
│   ├── (dashboard)/     # UI pages
│   └── api/             # REST endpoints
├── worker/               # Background worker process
│   ├── index.ts         # Entry point, starts pg-boss
│   ├── image-worker.ts  # Calls Runware API
│   └── video-worker.ts  # Calls Vertex AI Veo
├── lib/                  # Shared code
│   ├── db.ts            # Prisma client
│   ├── queue.ts         # pg-boss instance
│   └── storage.ts       # File save/read/zip helpers
├── prisma/schema.prisma  # DB schema
└── uploads/              # Generated files on disk
    ├── images/
    ├── videos/
    └── zips/
Request Flow
User pastes prompts → POST /api/projects/:id/generate
  → Inserts jobs into DB
  → Enqueues job IDs into pg-boss

Worker (always running):
  pg-boss fires job → image-worker or video-worker
  → Calls Runware / Vertex AI
  → Downloads & saves file to uploads/
  → Updates job status in DB

Frontend polls GET /api/projects/:id/status every 2s
  → Shows live progress bar
Dev startup (one command)
npm run dev  # runs Next.js + worker concurrently
